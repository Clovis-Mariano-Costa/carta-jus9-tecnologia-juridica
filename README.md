## Cuidados antes de alterar

1. Rodar `git status --short --branch`.
2. Preservar links, botoes e rotas antigas aprovadas.
3. Nao publicar tokens, chaves, senhas, `.env` real, dados pessoais sensiveis, documentos de cofre ou material sigiloso.
4. Conferir visualmente a pagina publicada apos deploy, especialmente acentuacao UTF-8, responsividade e links externos.
5. Submeter alteracoes sensiveis a revisao humana do Fundador.
